# 单细胞云平台 需求规格评审报告（2026-06）

> 评审人：项目负责人（兼开发）。评审对象：`docs/单细胞云平台-完善版需求规格说明书.md`。
> 方法：对照 `data/` 两个旧手工协作表的真实数据，8 个维度并行审查 + 每条发现独立对抗验证。
> 结论已据此修订规格；本报告留存供追溯「为什么这么改」。完整发现清单见 `tmp/audit_findings_clean.md`，事实基底见 `tmp/audit_facts.md`。

## 一、总体结论

文档骨架质量高、方向正确（旧 Excel 流水账 → 多对象 + 状态流转 + 克制地不上工作流引擎）。但「拿真实旧表一对照」暴露出一批按原文直接开发会卡死的问题。共确认 **81 个问题**（3 致命 / 11 重要 / 38 中等 / 29 低），另有 18 条经对抗验证判定为误读规格、予以驳回。已完成对致命与重要问题的修订。

## 二、老板已拍板的业务口径

1. **服务交付**：多数单子走生信分析后交付，少数只做到实验/质控/建库即交付，交付前可能需要质控 → 流程必须支持「含生信」「不含生信」两条交付路径。
2. **项目单位**：一个「项目」按**委托单**算（一个合同可对应多个项目）。
3. **历史脏数据**：先原样导入、保留原文，规范化清洗推后。
