# UI/UX 设计理念

> 本文是**判断框架**——遇到新界面/新交互时怎么决策。具体落地规则与被否决备选见 [ADR-0001](adr/0001-可见性全员开放与动作按替班分类.md)、[ADR-0002](adr/0002-详情整页化与查询参数工作面板落地.md)；视觉 token 见 `app/globals.css`。结论先行，理由附后。

## 1. 北极星与基调

- **对标当代主流产品，不照搬传统桌面/SaaS**。设计北极星是 **Vercel**（Dashboard 交互 + 视觉），辅以 Linear / Airtable / Stripe 等。拿不准时**先看主流怎么做**，不凭抽象推演。
- **视觉 = Vercel 式克制扁平**：灰底白面（页面/侧栏同为浅灰底，card/输入控件靠明度差浮起）、每区域只用一层边框（内部用分隔线/留白）、卡片与控件无投影（浮层保留）、状态用彩点 + 文字。
- **用户是队列驱动的内部员工**，不是外部客户。每个角色有自己的工位，关注的信息不同；系统目标是**提效**，不是"看起来正规"。

## 2. 决策原则（一个交互该长什么样，按这几条判断）

### 2.1 按「实体性质」选形态：容器 vs 叶子
- **容器 / hub**（包含子实体、可能有子页、用户要"驻留办公"）→ **整页**。如项目（含样本/任务/生信/交付/时间线）。
- **叶子 / 工作条目**（高频在队列里扫读和处理）→ **WorkItemPanel 工作项面板**。如实验任务/样本/生信任务。
- 依据：Vercel 的 Deployment（容器，有 Logs/Resources/Source 子页）点进**整页**；Log（叶子，高频扫读）点开**WorkItemPanel 工作项面板**。两种都用，按性质分。

### 2.2 按「任务性质」选形态：看 / 建 / 改是不同功能
- **查看详情、快速创建、编辑是三种不同任务**，不必共用同一种浮层。
- **一致性的正确含义**：同一功能跨页面要一致（所有"看叶子"一致、所有"创建"一致），**不是**把"看"和"建"两个不同功能强行统一成一种交互。Linear 的详情是整页、创建是模态——故意不同。
- 当前映射：叶子**工作项面板** = `?view` 工作项面板（状态驱动：待处理直接呈现主任务表单，已处理呈现摘要与修正入口，不放右侧时间线）；叶子**快速创建** = 居中模态（`?new`，从列表建一条记录的主流范式，Linear/Airtable）；**状态推进** = 语义动作表单；**容器/重表单创建** = 整页。

### 2.3 行内动作优先，详情才能"轻"
- 快捷操作放在**列表行尾**（语义动作按钮 + 破坏性动作收 `⋯`），**不必进详情即可干活**。
- 正因如此，"详情"可以是轻量浮层、甚至整页，都不会逼用户"为了操作而层层跳转"。**禁止**回退到"动作只能进详情页做"。

### 2.4 常驻侧栏 + 面包屑 = 不丢上下文
- 有了**常驻侧栏**（上下文锚点）+ **顶栏面包屑**（一键回列表）+ 返回时**列表筛选保留**（在 URL 里），**整页详情也不"丢列表上下文"**。
- 所以传统 "list→detail→action 每条 4 跳丢上下文" 的老痛点，是被**行内动作 + 常驻工作项面板**消解的，而非靠"详情必须做成滑出面板"。

### 2.5 用查询参数驱动浮层，不用拦截路由
- 工作区/模态用 **URL 查询参数**（`?view=`、`?new=`）驱动：可分享、返回键关闭、关闭只删参保留筛选。
- **不用** Next.js 拦截路由 + `@sheet` 平行槽——官方但脆、过度（`(.)`/平行槽/`/new` 撞 `(.)[id]` 都踩过），且其"同 URL 双形态"我们不需要（工作项面板是处理面板，整页是目的地，本就该不同 URL）。面板实现仍用 shadcn `Sheet` 组件（这是普通 UI 组件，并非 Next.js `@sheet` 平行槽）。

## 3. 当前落地形态（速查，详见 ADR-0002）

| 功能 | 形态 |
|---|---|
| 容器（项目）详情 | 整页 + 顶栏面包屑 |
| 叶子工作项（实验/样本/生信） | `?view` WorkItemPanel（主任务表单或摘要 + 修正入口，无右侧时间线） |
| 叶子快速创建 | `?new` 居中模态（含上下文预填 `?new&parentId=`） |
| 叶子属性编辑 | 工作区分区编辑（读态展示，编辑态整段表单，一次保存） |
| 项目创建 / 项目编辑 | 整页表单 |
| 列表行动作 | 行尾语义动作 + `⋯` 溢出 |
| 交叉链（A 点到 B） | 跳目标工位/整页，不叠跨实体浮层 |

顶栏面包屑由页面经 context 送入（`SetBreadcrumb` → `HeaderTitle`），不渲染在正文（免与大标题重复）。

## 4. 列表页范式

- **单行工具栏**：搜索即时生效（控件白底）+ 筛选；**多选状态筛选**、**默认隐藏终态**（聚焦在途，终态一键可见）。
- **白底圆角边框表格**，行 flush 到容器边；统一 `ListPager` 页脚；**可操作空态**（区分"无匹配"与"暂无"，默认隐藏终态时空态点明"点筛选看全部"）。
- **工位 = 角色队列**：每个指标全站**只出现一次**；态势感知靠角色落地页 + 队列/`awaiting`/需关注筛选 + 侧栏角标，不靠藏行。

## 5. 权限与可见性（见 ADR-0001）

- 内部提效系统、全员可信：**可见性全员开放**（不按角色藏行）；"角色差异"用于**工位聚焦**，不是访问控制。
- **动作权限按"能否替班"分类**：操作类（谁在干谁记）放开给全员在岗，管理类（确认/交付/异常/删除）仍锁 admin/PM。

## 6. 反模式（别做）

- 别因"一致"把**看/建/改**强行统一成一种浮层——它们是不同功能。
- 别回退到"**动作只能进详情页做**"。
- 别用**拦截路由**做浮层（脆、过度）；浮层用查询参数。
- 别套传统**三栏常驻 / 重审批 / 重权限** SaaS 范式（内部提效系统）。
- 别**凭抽象推演**定交互——先看 Vercel/Linear 等主流怎么做。
- 别让**指标在多处重复**出现；别用 `bg-background`/`bg-transparent` 冒充"白色"。
