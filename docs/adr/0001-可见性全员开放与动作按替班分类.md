# ADR-0001 可见性全员开放 + 动作按"能否替班"分类

- 状态：已采纳（2026-06-29）
- 取代：规格说明书 §3.3「数据可见范围（行级 scope）」（该节作废）
- 决策者：产品负责人（老板）拍板，AI 实现
- 相关代码：`lib/auth/role-scope.ts`、`lib/auth/action-roles.ts`、各 `lib/*/rules.ts` 与 `service.ts`

## 背景

规格 §3.3 把「行级数据可见范围」定为**贯穿数据层的安全横切约定**：所有列表/详情/导出/统计按角色自动注入 Prisma `where`，销售只见自己负责、实验员只见相关任务的项目等。

但本系统是**公司内部的提效工具**——全员是可信同事、无外部租户、无客户门户。实践中，为"看起来正规的权限"按角色藏行带来的是纯摩擦而非安全收益：

- 状态机能流转，但"子实体还没被创建、要人手动建"的接缝处，下一棒角色的工作面里没有可显示的东西。为绕过这点，被迫给 `buildSampleScope`/`buildExperimentTaskScope`/`buildBioinfoTaskScope` 并上"待建池"例外（鸡生蛋补丁）。
- 生信分析员看未认领任务时，项目 scope 恒假 → 详情页查询 404 拖垮整页，只能 `.catch(()=>[])` 兜底。

这些补丁都是"墙太严 → 打断协作 → 再补丁"的循环，墙本身是病根。

## 决策

把"角色差异"的目的从**访问控制（藏行）**改为**工位聚焦（把对的信息推到对的人面前）**，二者对内部提效系统价值不同：

1. **可见性全员开放**：`build*Scope` 对任何在册角色（含只读 `viewer`）返回空 `where`，仅未知/缺失角色 `{ id: "__none__" }` fail-closed。列表/详情/导出/统计/徽章一律全量，敏感字段也都给。徽章退化为**团队待办计数**（谁都能接手）。`build*Scope` 第二参 `userId` 删除（不再按人过滤）。"工位聚焦"改由正交机制承担：角色落地页 + 队列/`awaiting`/需关注筛选 + 侧栏徽章——不靠藏行。

2. **动作权限按"是否真需要权限且不能替班"分两类**，与可见性正交：
   - **仍锁 admin/PM**（管理决策、有问责、不能随便替）：确认项目 / 确认交付 / 标记·恢复·终止异常；删除项目 = 仅 admin；建·改项目头 = `projectOwnerRoles`(admin/PM/销售)。
   - **放开给全员在岗**（`staffRoles` = 除 `viewer`，判定函数 `canActAsStaff` 用白名单）：收样 / 实验全流程（排期·开始·完成·反馈·质控）/ 产出指标 / 生信建单与执行 / 多模态确认 / 测序交付登记。原"必须本人才能操作该条"的实例归属守卫（已退化为与入口 `ensure*Role` 同检查）一并删除。操作类角色常量（`experimentManageRoles`/`bioinfoManageRoles`/`bioinfoCreateRoles`/`sampleReceiveRoles`/`sampleCreateRoles`/`metricsRecordRoles`）统一指向 `staffRoles`，名为 manage/create 仅兼容历史调用点。
   - **viewer 保持只读**（动作入口的 `ensure*Role(staffRoles)` 将其挡在外）。

## 被否决的备选

- **维持 §3.3 行级安全墙**：对全员可信的内部提效系统是纯摩擦、无真实安全收益，且已被一堆补丁证伪。
- **角色化的可见性 / 需关注集**：给每个角色配不同的藏行规则或不同的"需关注"状态——为边际收益加复杂度，过度设计。

## 影响

- 移除"鸡生蛋池例外"（`samplesAwaitingTaskWhere`/`tasksAwaitingBioinfoWhere` 不再进 scope，仅保留为列表 `awaiting=` 筛选与徽章口径）。
- 移除冗余实例归属守卫 `ensureCanOperateTask`/`ensureCanOperateBioinfoTask`。
- `getWritable*` 前置读不带 scope（已加注释说明，防止误加回 scope 重新引入限制）。
- 测试按新契约重写：可见性（全员开放 + 未知角色 fail-closed）、动作权限（在岗可做、viewer 不可）。
- **规格 §3.3 作废**（正文已加作废横幅指向本 ADR）；CLAUDE.md 保留 1 行指针 + 核心不变量。

## 验证

`tsc` / `lint` / 177 单测 / `build` 全绿。多角色真实环境实测：lab/viewer 与 admin 可见数据相同；详情页未指派项目 404→200；管理动作（lab 确认项目）403；操作动作（sales 排期/提交）非 403；viewer 全部 403。
