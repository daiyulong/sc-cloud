# ADR-0004 生信工位负责人维度重整：移除 range 分段 + 「离开 pending 必有负责人」不变量

- 状态：已采纳（2026-07-01）
- 关联：延续 [ADR-0001](0001-可见性全员开放与动作按替班分类.md)（开放可见 + 动作按"能否替班"分类）；与 [ADR-0003](0003-工位列表状态tab与scope-toggle正交化.md) 同「工位列表 scope 移除」模式
- 相关代码：`app/(app)/bioinfo-tasks/page.tsx`、`lib/schemas/bioinfo-task.ts`、`lib/bioinfo-tasks/{service,rules}.ts`、`components/bioinfo-tasks/bioinfo-task-primary-action.tsx`

## 背景

生信工位顶部有「我的 / 待分配 / 全部」`range` 分段控件（`?range=mine|unassigned|all`）。审视其三个值：

- `mine`（`analystId = 我`）、`all`（无过滤）是**纯 scope** 切换——和 [ADR-0003] 已在 `/lab` 移除的「我的/团队」scope toggle 同类。开放可见模型下（ADR-0001）个人/团队视角切换价值低于复杂度，是历史遗留。
- `unassigned`（`analystId = null 且 status = pending`）是**有实义的工作队列**（没人认领的待开始任务），但被裹在 range 里。

同时这块暴露了一个**状态机缺口**：`status`（工作流进度）与 `analystId`（谁负责）当前完全正交、零约束。`startBioinfoTask`（pending→in_progress）原本只有 `if (input.analystId !== undefined)` 才动负责人，而 UI「开始分析」按钮不带负责人 → 可产生 **「分析中（in_progress）+ 未分配（analystId=null）」** 这种"没人负责却在做"的自相矛盾态，且无任何校验收敛。

## 决策

1. **移除 `range` 分段**：`schema` / `service` where / `page` 删除 `range`（mine/unassigned/all）。默认列表统一为「全部 + 隐藏终态」（`DEFAULT_BIOINFO_STATUS_SELECTION`），不再按角色给默认 scope（原 `bioinfo_analyst` 默认落「我的」取消）。scope 深链接能力由 `?analystId=` 承担（不在 UI 常驻暴露，同 `/lab` 的 `operatorId`）。

2. **负责人维度改由 `ListToolbar` 单选菜单承载**：新增「分析负责人」菜单 = 全部负责人 + 各在岗分析员 + **「未分配」哨兵项**（`analystId=unassigned`，后端 `buildBioinfoListWhere` 映射为 `{ analystId: null }`；哨兵常量 `ANALYST_UNASSIGNED` 前后端共用，不散魔法串）。「看我的」= 在菜单里选自己（开放可见下也能看别人，不再是写死的 scope）。

3. **补状态机不变量：离开 `pending` 必有负责人**（`in_progress` 及以后 `analystId ≠ null`）。`start` 时定负责人优先级：**显式指定 > 任务已有 > 分析员本人自动认领（谁开始谁负责）**；非分析员（PM 等）代推进且任务未分配时报错，要求先指定。逻辑抽为纯函数 `resolveStartAnalyst`（`rules.ts`，可单测）；显式指定者由 `ensureActiveAnalyst` 校验为在岗 `bioinfo_analyst`（防 API 塞任意 userId）。
   - 前端「开始分析」据此分流：已分配 / 分析员本人 → 直接开始（后端自动认领）；未分配 + 非分析员 → 先选负责人再开始。
   - 于是「未分配」筛选语义**收敛为「pending 里还没人认领的」**——补回 range 时代 `unassigned` 队列的用途，但更规整（不再需要 `null + pending` 的组合筛，也不会混入异常态）。

## 被否决的备选

- **保留 `range` 分段**：`mine`/`all` 是 ADR-0001 已判定为纯摩擦的 scope；`unassigned` 的队列语义用「分析负责人 = 未分配」更通用地覆盖。
- **维持 `status × analystId` 正交、允许「分析中 + 未分配」作合法中间态**：承认"没人负责却在分析中"是数据噪声，会让「未分配」筛选混入异常态，且与 ADR-0001「谁做谁负责」精神相悖。
- **`start` 一律弹表单强制选负责人（含分析员本人）**：分析员本人开始时多一步手选自己，不如自动认领省事；仅在"非分析员代推进未分配任务"时才需要显式选。

## 影响

- 生信工位顶部无常驻切换控件，与收样/样本工位一致。实验工位保留「任务/排期」胶囊——它真有两种视图（列表 vs 排期看板），属**有理由的差异**，非历史不一致。
- **存量**：新契约只在 `start` 时生效，不回填历史「分析中 + 未分配」任务；`seed` 不建生信任务，fresh 环境无此存量。若某演示部署有旧数据，其历史态不受影响。
- 测试：`tests/bioinfo-task-list-where.test.ts`（analystId 过滤含 `unassigned`→`null`）、`tests/bioinfo-task-rules.test.ts`（`resolveStartAnalyst` 各分支）、`tests/bioinfo-task-schema.test.ts`（移除 `range` 断言）。
