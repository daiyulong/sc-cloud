# ADR-0002 详情交互 = 整页(容器) + `?view` 工作项面板(叶子)，弃用拦截路由

- 状态：已采纳（2026-06-29）；2026-06-30 修订：叶子详情收敛为状态驱动的 `WorkItemPanel`；删除 WorkspaceSheet 右栏时间线、列表动作右抽屉、叶子全页兼容跳转；2026-07-01 修订：项目工位内复用同一 `WorkItemPanel`（不再跨页跳走），`?view` 收尾契约统一由 `useCloseWorkItemPanel` 承担
- 取代：CLAUDE.md「UI 交互模型」中**详情用侧滑（@sheet 拦截路由）**的原方案
- 决策者：产品负责人（老板）拍板，AI 实现
- 相关代码：`components/detail/work-item-panel.tsx`、`components/detail/editable-section.tsx`、`components/list/clickable-row.tsx`、各工位列表页

## 背景

原方案用 Next.js **拦截路由 + 平行路由（`@sheet` 槽）** 实现"软导航开侧滑、同 URL 硬加载开全页"。这是官方认证模式，但：

- **复杂且脆**：`(.)`/`(..)` 约定反直觉、要 `default.tsx` + catchAll 兜平行槽；踩过 `/new` 撞 `(.)[id]` 冲突（曾要静态压制）。
- **抽屉装不下项目 hub**：项目详情含样本/任务/生信/交付/时间线，侧滑抽屉过挤。
- **in-app 到不了全页**：任何地方点实体都进侧滑，没有干净路径到全页详情；全页又无面包屑/返回。

**Vercel 实证（设计北极星）**：Deployment（容器，有 Logs/Resources/Source 子页）点击进**整页 + 面包屑**；Log（叶子，高频扫读）点击开**右抽屉 + ▲▼ 上/下一条**。两种都用，按实体性质分。

**关键再判断**：原"侧滑保列表上下文"的 WHY，已被**行内动作（快捷操作不必进详情）+ 常驻侧栏 + 面包屑**覆盖——侧滑当初要解决的痛点已不存在。且"抽屉/整页同一 URL"这个拦截路由独有属性，我们其实不需要（工作区是列表上下文内处理叶子对象、整页是独立目的地，本就该不同 URL）。

**2026-06-30 再修订**：样本接收迭代后出现旧详情抽屉、登记接收抽屉、工作区内动作模态、分区编辑并存，同一业务意图有多入口，心智负担过重。最终收敛为 `WorkItemPanel`：它不是“详情容器”，而是列表上下文内处理一条叶子工作项的单栏面板。待处理状态直接呈现当前主任务表单（如待到样直接登记接收）；已处理状态才呈现摘要与修正入口。不内置右侧时间线，不保留列表动作右抽屉，不保留叶子全页兼容跳转。

## 决策

**判据：容器/hub 实体 → 整页 + 面包屑；叶子工作对象 → WorkItemPanel。**

1. **项目 = 整页**（`/projects/[id]`，顶部面包屑「项目 / 编号」）。
2. **实验任务 / 样本 / 生信任务 = `?view=<id>` 查询参数工作项面板**：在各自工位列表（`/lab`、`/intake`、`/bioinfo-tasks`）渲染 `WorkItemPanel`；行/编号点击叠加 `?view` 参数、关闭只删该参数保留筛选；**不用拦截路由**。列表行主动作也只打开同一个面板，不直接弹动作抽屉。
3. **项目内也挂同一个 `WorkItemPanel`（不跨页跳走）**：`/projects/[id]` 表内直接渲染 `WorkItemPanel`，关掉留在原 Tab；项目内交叉链（A 实体的 Tab 中看到 B 实体）不再甩出站、点一行直接打开本地工作项面板，避免上下文丢失。"跳目标工位"按钮（卡片右上「在 X 工位查看」）作为深钻入口保留但不再是默认入口。
4. **表单分三类**：① **叶子创建（实验/生信任务）= `?new` 居中模态**——队列里高频建，成功后关模态回列表、不跳全页。② **当前状态的主任务 = WorkItemPanel 内嵌表单**——如待到样打开即登记接收，不再先看详情再弹二级表单。③ **已处理后的信息修正 = 分区编辑**——读态展示，点“修正信息”后整段变表单，一次保存。项目创建和项目编辑仍为整页表单。
5. **拆除整个 `@sheet` 平行槽 + 拦截路由**：P1 删 4 个详情拦截 + `project-sheet-body`（项目走全页不用它）；P2 删 3 个 create 拦截 + 槽 + `default`/catchAll + `FormModal`。

## 被否决的备选

- **维持拦截路由侧滑**：脆、过度，且其"同 URL 双形态"属性我们不需要。
- **全部整页**（连叶子也整页）：叶子高频扫读和处理需要保留工位列表上下文，全整页丢掉它。
- **全部工作区**（连项目也工作区）：项目 hub 内容太重，工作区装不下，应是整页。
- **叶子创建也走全页**（P2 初版如此）：实测打断队列工作流——待建里一条条建、每次被甩出队列再返回，正是行内动作要避免的"丢上下文"；改为 `?new` 居中模态（项目创建/编辑仍整页）。
- **叶子创建用右抽屉**（曾短暂实现，想与详情 `?view` 抽屉一致）："一致性"是同一功能跨页要一致、非把"看/建"两个不同功能强行统一；从列表快速建一条记录的主流是**居中模态**（Linear/Airtable），且两列表单在窄抽屉偏挤。故用居中模态 `FormDialog`。
- **列表动作右抽屉**：和 WorkItemPanel 形成两套处理入口，同一动作在不同位置呈现为不同浮层，删除。
- **叶子全页兼容跳转**：保留旧 URL 会暗示仍存在第二种详情/编辑路径，删除。

## 影响

- `WorkItemPanel` 查询参数驱动（关闭 = `router.push` 删 `view` 参数，非 `router.back`）；旧详情抽屉组件删除，不保留第二套叶子详情入口。
- 叶子全页详情/编辑路由删除；前端链接统一指向对应工位 `?view=id`。
- 分区编辑只调用既有 `PUT /api/*/[id]`，状态推进仍调用显式动作 API；当前主任务表单直接内嵌到 WorkItemPanel。
- **顶栏面包屑**（`components/header-breadcrumb.tsx`）：详情页 `SetDetailBreadcrumb` 经 context 把「上级/当前」送进顶栏 `HeaderTitle`（无面包屑时回退段标题），**不渲染在正文**（避免与正文大标题重复）；多级（末段无 href=当前页），4 个详情 + 7 个表单（新建/编辑）全页均接入。`ClickableRow` 加 `scroll`（工作项面板打开不滚顶）。
- 工作项面板测试按新关闭契约重写。
- **▲▼ 上/下一条**（Vercel Logs 同款）列为后续增强，本次未做。
- P2 拆除：`@sheet` 平行槽（含 3 个 `(.)entity/new` 拦截 + `default`/catchAll）、`FormModal`、layout 的 `sheet` 槽；叶子「新建」按钮统一改为 `?new` 居中模态，项目创建仍为整页。
