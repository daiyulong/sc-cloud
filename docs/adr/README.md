# 架构决策记录（ADR）

记录**不可从代码反推**的重大设计决策：被否决的备选、踩坑得来的参数、跨系统的非显式契约、暂时性妥协、外部约束，尤其是**反转既有决策**的场景。

约定：

- 一决策一文件，命名 `NNNN-简短标题.md`（四位序号递增）。
- 每篇含：状态 / 背景 / 决策 / 被否决的备选 / 影响。事实标注证据（commit、文件），推断标注"未验证"。
- 决策落地后，CLAUDE.md 只留 1 行指针（`见 ADR-NNNN`）+ 最核心的不变量；完整记录在 ADR。
- 旧决策被取代时，新 ADR 写明 `取代：ADR-XXXX / 规格 §X`，并在被取代处加指针，不删除历史。
