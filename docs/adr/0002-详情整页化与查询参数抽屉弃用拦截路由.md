# ADR-0002 详情交互 = 整页(容器) + `?view` 抽屉(叶子)，弃用拦截路由

- 状态：已采纳（2026-06-29）；P1（详情整页/抽屉）+ P2（拆 `@sheet` 槽）+ 叶子创建 `?new` 居中模态（实测修订"表单一律整页"）均已落地
- 取代：CLAUDE.md「UI 交互模型」中**详情用侧滑（@sheet 拦截路由）**的原方案
- 决策者：产品负责人（老板）拍板，AI 实现
- 相关代码：`components/detail/detail-sheet.tsx`、`detail-breadcrumb.tsx`、`components/list/clickable-row.tsx`、各工位列表页、各 `[id]` 全页详情

## 背景

原方案用 Next.js **拦截路由 + 平行路由（`@sheet` 槽）** 实现"软导航开侧滑、同 URL 硬加载开全页"。这是官方认证模式，但：

- **复杂且脆**：`(.)`/`(..)` 约定反直觉、要 `default.tsx` + catchAll 兜平行槽；踩过 `/new` 撞 `(.)[id]` 冲突（曾要静态压制）。
- **抽屉装不下项目 hub**：项目详情含样本/任务/生信/交付/时间线，侧滑抽屉过挤。
- **in-app 到不了全页**：任何地方点实体都进侧滑，没有干净路径到全页详情；全页又无面包屑/返回。

**Vercel 实证（设计北极星）**：Deployment（容器，有 Logs/Resources/Source 子页）点击进**整页 + 面包屑**；Log（叶子，高频扫读）点击开**右抽屉 + ▲▼ 上/下一条**。两种都用，按实体性质分。

**关键再判断**：原"侧滑保列表上下文"的 WHY，已被**行内动作（快捷操作不必进详情）+ 常驻侧栏 + 面包屑**覆盖——侧滑当初要解决的痛点已不存在。且"抽屉/整页同一 URL"这个拦截路由独有属性，我们其实不需要（抽屉是"在列表上瞥一眼"、整页是独立目的地，本就该不同 URL）。

## 决策

**判据：容器/hub 实体 → 整页 + 面包屑；叶子工作条目 → 抽屉。**

1. **项目 = 整页**（`/projects/[id]`，顶部面包屑「项目 / 编号」）。
2. **实验任务 / 样本 / 生信任务 = `?view=<id>` 查询参数抽屉**：在各自工位列表（`/lab`、`/intake`、`/bioinfo` 进行中）渲染 `DetailSheet`；行/编号点击叠加 `?view` 参数、关闭只删该参数保留筛选；**不用拦截路由**。抽屉内 ⤢『在全页中打开』（各 `*SheetBody` 的 `Maximize2`）可升级到对应全页 hub——`DetailSheet` 不另加重复入口。
3. **交叉链 → 整页跳转**（不叠跨实体浮层）：待建里的实验任务、`/experiences`、项目↔子实体之间。"全局把任意实体叠成浮层"是拦截路由的能力，在本模型里用不上。
4. **表单分两类（实测修订）**：① **叶子创建（实验/生信任务）= `?new` 居中模态**——队列里高频建（待建 tab 一条条建），全页把人甩出队列、打断工作流；用 `?new=1` 居中模态（`FormDialog`、表单传 `surface="modal"`）留在队列，成功后关模态回列表、不跳全页，含 `?new=1&experimentTaskId=X` 上下文预填。模态而非右抽屉：从列表快速建一条记录是主流模态范式（Linear/Airtable），且"看/建"是不同功能、无需共用同一种浮层。② **项目创建 + 所有编辑 = 整页**（`/entity/new`、`/entity/[id]/edit`）——项目是根、表单大、刻意动作；编辑从详情进、一次性、不在队列批量做。拦截已删、`FormModal` 随之删除。
5. **拆除整个 `@sheet` 平行槽 + 拦截路由**：P1 删 4 个详情拦截 + `project-sheet-body`（项目走全页不用它）；P2 删 3 个 create 拦截 + 槽 + `default`/catchAll + `FormModal`。

## 被否决的备选

- **维持拦截路由侧滑**：脆、过度，且其"同 URL 双形态"属性我们不需要。
- **全部整页**（连叶子也整页）：叶子高频扫读，抽屉 + ▲▼ 连续翻看更顺（Vercel Logs 体验），全整页丢掉它。
- **全部抽屉**（连项目也抽屉）：项目 hub 内容太重，侧滑装不下。
- **叶子创建也走全页**（P2 初版如此）：实测打断队列工作流——待建里一条条建、每次被甩出队列再返回，正是行内动作要避免的"丢上下文"；改为 `?new` 居中模态（项目创建/编辑仍整页）。
- **叶子创建用右抽屉**（曾短暂实现，想与详情 `?view` 抽屉一致）："一致性"是同一功能跨页要一致、非把"看/建"两个不同功能强行统一；从列表快速建一条记录的主流是**居中模态**（Linear/Airtable），且两列表单在窄抽屉偏挤。故用居中模态 `FormDialog`。

## 影响

- `DetailSheet` 改查询参数驱动（关闭 = `router.push` 删 `view` 参数，非 `router.back`）；升级全页复用各 `*SheetBody` 既有的 ⤢『在全页中打开』，不另加重复入口。
- **顶栏面包屑**（`components/header-breadcrumb.tsx`）：详情页 `SetDetailBreadcrumb` 经 context 把「上级/当前」送进顶栏 `HeaderTitle`（无面包屑时回退段标题），**不渲染在正文**（避免与正文大标题重复）；多级（末段无 href=当前页），4 个详情 + 7 个表单（新建/编辑）全页均接入。`ClickableRow` 加 `scroll`（抽屉打开不滚顶）。
- 详情侧滑测试按新关闭契约重写。
- **▲▼ 上/下一条**（Vercel Logs 同款）列为后续增强，本次未做。
- P2 拆除：`@sheet` 平行槽（含 3 个 `(.)entity/new` 拦截 + `default`/catchAll）、`FormModal`、layout 的 `sheet` 槽；「新建」按钮自动变整页。
